# Coding Standards & Best Practices

- **Strict Typing**: TypeScript code MUST be strictly typed.
  - NEVER use `any`.
  - NEVER use `as unknown`.
  - NEVER use `never`.
- **Clean Code**: Follow Clean Code principles. Code must be readable, maintainable, and well-structured.
- **Reusability**: Always prefer creating reusable files, utilities, or components rather than duplicating code. Check if a utility already exists before creating a new one.
- **Knowledge & Skills Review**: ALWAYS check the `.ai-knowledge.md` file and the `.agents/skills` (or `.skills`) directory for existing domain knowledge, workflows, and architecture instructions before starting a task.
